프로젝트 주제 : 디지털 플래너 (스케쥴 관리 시스템)

1. 개요

 1) 목적
  - 개인의 일정 관리와 주소록 관리가 한 눈에 가능한 디지털 플래너를 개발하는 것이 주 목적이다.
  - 손에 들고 다니는 다이어리를 디지털화하고 기존에 있는 일정 관리 프로그램들을 개선해 새로운 다이어리, 플래너 시스템을 만들려고 한다.
  - 일반적인 일정 관리에서 벗어나 간트 차트나 그래프와 같은 시각적인 효과를 제공해 사용자가 일정을 관리하는데 있어서 보다 편리하게 확인을 할 수 있다.
  - 주소록 기능을 추가해 연락처 관리를 용이하게 할 수 있고 전화번호, 메일 주소 등을 함께 포함해 바로 연락이 가능하도록 제공한다.
  - 가능하다면 사용자 간의 직접적인 채팅 기능을 제공해 필요시 바로 연락이 가능하도록 한다.
  - 깃허브의 이슈와 프로젝트 메뉴와 같은 기능을 제공해 팀프로젝트 관련 기능도 제공한다.
  - 

 2) 주요 사용자 및 지원 서비스
  - 일정 관리를 필요로 하는 사용자


2. 프로젝트 설계 계획

 1) 설계 범위
  - 
  - 사용할 언어 : Java, MySQL, JDBC, HTML5, CSS3, Java Script

 2) 설계 과정
  - 요구 분석 및 제약 정의 : 프로젝트의 주제를 정확하게 파악하고 구현할 기능 등을 정한다. 개발에 있어서 제약 사항을 파악하여 프로젝트 진행에서 예외상황을 최대한 줄인다.
  - 프로토타입 제작 : 완성될 시스템의 프로토타입을 제작하여 프로젝트의 전체적인 틀과 기준을 정한다.
  - 시스템 구조 설계 : 프로토타입을 참고해 시스템의 전체적인 구조를 설계한다.
  - 상세 기능 및 데이터베이스 구조 설계 : 시스템 구조를 참고해서 필요한 상세 기능들을 정하고 데이터베이스 스키마 등의 구조를 설계한다.
  - 상세 기능 구현 및 데이터베이스 구현 : 코딩을 통해 시스템의 상세 기능과 데이터베이스를 구현한다.
  - UI 설계 및 수정 : 구현한 시스템과 프로토타입을 이용해 UI를 설계 및 수정한다.
  - 테스트 및 디버깅 : 완성된 시스템으로 테스트와 디버깅과정을 진행하고 최종 완성한다.

4. 제한 사항
 1) 본 프로젝트는 수업에서 배운 내용들을 최대한 많이 사용할 수 있도록 프로젝트를 선정한다.
 2) 팀원들의 능력치를 감안하여 구현이 가능한 것과 그렇지 못한 것을 구체적으로 정해야 한다.
 3) 기본적인 사용 환경은 PC를 중심으로 진행되어야 하며, 어느 한 플랫폼에 국한되는 것을 최대한 자제한다.
 4) 프로젝트는 완성을 1순위 목표로 하고 완성을 위해선 제안서의 수정도 불가피하다.
 5) 시스템은 Java를 이용한 프로그래밍을 기초로 하고 웹페이지 제작, 데이터베이스 연동 모두를 구현하는 것을 기본으로 한다.
 6) 시스템의 기능은 4 ~ 5개 정도로 제한하고 팀원의 능력과 일정 계획에 따라서 추가 또는 삭제를 할 수도 있다.
