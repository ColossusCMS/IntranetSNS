프로젝트 주제 : 학생 관리 시스템

1. 개요

 1) 목적
  - 학생 리스트, 출결 관리, 성적 관리 등의 정보를 관리할 수 있는 학생 관리 시스템을 개발하는 것이 주목적이다.
  - 학생 정보 관리 뿐만 아니라 수업의 일정과 공지사항 등록으로 학생들에게 수업과 관련된 정보를 제공할 수 있고 학생들은 개개인의 계정으로 본인의 출결 현황, 성적 확인, 수업의 일정확인과 공지사항 확인 등이 가능하다.
  - 또한 해당 수업의 고유한 게시판을 제공해 질문사항이나 의견을 작성할 수도 있고 선생님 또는 다른 학생들과의 커뮤니티 용도로 사용한다.

 2) 주요 사용자 및 지원 서비스
  - 관리자 : 관리자는 고유 계정을 가지고 있고 학생의 등록과 삭제, 정보 수정이 가능하다.
게시판의 개설과 관리가 가능하다. 
  - 선생님 : 선생님 전용 계정이 존재하며 학생 출결 관리, 성적 관리, 공지사항 등록, 수업 일정 관리 등이 가능하다. 그리고 게시판 사용이 가능하다.
  - 학생 : 학생 전용 계정이 존재하며 학생마다 고유한 계정을 가진다. 학생은 본인의 출결 현황, 성적 확인, 수업 일정 확인, 공지사항 확인이 가능하다. 그리고 게시판 사용이 가능하다.


2. 프로젝트 설계 계획

 1) 설계 범위
  - 반응형 웹페이지를 기반으로 제작한다.
  - 반응형 웹페이지를 통해 PC 뿐만 아니라 모바일 환경을 함께 제공하여 스마트폰 또는 태블릿으로도 언제든지 접속할 수 있도록 하는 것이 목표이다. 
  - 데이터베이스를 이용해 학생들의 상세 정보와 각 계정들의 정보를 저장한다.
  - 사용할 언어 : Java, Java Script, HTML5, CSS3, JSP, MySQL, JDBC 

 2) 설계 과정
  - 요구 분석 및 제약 정의 : 프로젝트의 주제를 정확하게 파악하고 구현할 기능 등을 정한다. 개발에 있어서 제약 사항을 파악하여 프로젝트 진행에서 예외상황을 최대한 줄인다.
  - 프로토타입 제작 : 완성될 시스템의 프로토타입을 제작하여 프로젝트의 전체적인 틀과 기준을 정한다.
  - 시스템 구조 설계 : 프로토타입을 참고해 시스템의 전체적인 구조를 설계한다.
  - 상세 기능 및 데이터베이스 구조 설계 : 시스템 구조를 참고해서 필요한 상세 기능들을 정하고 데이터베이스 스키마 등의 구조를 설계한다.
  - 상세 기능 구현 및 데이터베이스 구현 : 코딩을 통해 시스템의 상세 기능과 데이터베이스를 구현한다.
  - UI 설계 및 수정 : 구현한 시스템과 프로토타입을 이용해 UI를 설계 및 수정한다.
  - 테스트 및 디버깅 : 완성된 시스템으로 테스트와 디버깅과정을 진행하고 최종 완성한다.
